# Radical Candor Framework

## 개요

Radical Candor는 Kim Scott이 개발한 피드백 프레임워크입니다.
"Care Personally"와 "Challenge Directly"의 균형을 통해 효과적인 피드백 문화를 만듭니다.

## 핵심 개념

### 🎯 2x2 Matrix

```
         Challenge Directly →
    ↑   ┌────────────┬────────────┐
    │   │            │            │
Care    │   Ruinous  │  Radical   │
Person- │   Empathy  │   Candor   │
ally    │            │            │
    │   ├────────────┼────────────┤
    │   │            │            │
    │   │Manipulative│  Obnoxious │
    │   │ Insincerity│ Aggression │
    │   │            │            │
    └   └────────────┴────────────┘
```

### 📊 4가지 영역

**Radical Candor (급진적 솔직함)** ✅
- Care Personally + Challenge Directly
- 진심으로 신경 쓰면서 직접적으로 피드백
- 성장과 신뢰의 선순환

**Ruinous Empathy (파괴적 공감)** ⚠️
- Care Personally + 도전 부족
- 상처 주기 싫어서 피드백 회피
- 단기적 편안함, 장기적 해악

**Obnoxious Aggression (불쾌한 공격)** ❌
- Challenge Directly + 관심 부족
- 무례하고 공격적인 비판
- 관계 파괴, 방어적 반응

**Manipulative Insincerity (교묘한 불성실)** ❌
- 관심도 도전도 없음
- 정치적이고 수동공격적
- 신뢰 완전 붕괴

## Care Personally

### 💙 개인적 관심의 의미

**Whole Person 인정**
- 업무 수행자가 아닌 한 사람으로
- 개인 삶과 가치관 존중
- 성장 욕구 이해

**Genuine Relationship**
- 진정성 있는 관계 구축
- 상호 신뢰 기반
- 장기적 관점

### 🛠 실천 방법

**1-on-1 미팅**
- 정기적 시간 확보
- 개인 이야기 경청
- 커리어 목표 논의

**일상적 관심**
- 작은 것 기억하기
- 성과 인정하기
- 어려움 공감하기

**팀 문화**
- 심리적 안전감 조성
- 실수 허용
- 다양성 존중

## Challenge Directly

### 🎯 직접적 도전의 의미

**Clear & Specific**
- 모호함 없는 명확한 피드백
- 구체적 사례와 영향
- 개선 방향 제시

**Timely**
- 즉시 또는 빠른 시일 내
- 기억이 생생할 때
- 반복 전에 개입

**Helpful Intent**
- 성장을 위한 피드백
- 문제 해결 초점
- 미래 지향적

### 📝 피드백 구조

**SBI Model**
```
Situation: 언제, 어디서
Behavior: 관찰된 행동
Impact: 결과와 영향
```

**예시**
```
S: "어제 미팅에서"
B: "클라이언트 질문에 준비 없이 답변했을 때"
I: "신뢰도가 떨어져 보였고, 계약에 영향 줄 수 있어요"
```

## Radical Candor 실천

### 🗣 대화 기법

**Praise 주기**
- Public & Specific
- 즉시 인정
- 진심 담기
- 영향 설명

**Criticism 주기**
- Private & Humble
- 개선 가능성 믿기
- 대화로 진행
- 해결책 함께 찾기

### 💬 대화 예시

**Radical Candor ✅**
> "발표 준비를 정말 열심히 한 게 보였어요. 다만 슬라이드 텍스트가 너무 많아서 핵심이 흐려졌어요. 다음엔 키 메시지 3개로 줄이면 훨씬 임팩트 있을 거예요."

**Ruinous Empathy ❌**
> "발표 잘했어요! 정말 좋았어요!" (실제로는 개선점이 많았음)

**Obnoxious Aggression ❌**
> "슬라이드가 엉망이었어. 이렇게 하면 누가 이해하겠어?"

**Manipulative Insincerity ❌**
> "음... 괜찮았어요." (뒤에서 다른 사람에게 불평)

## PM을 위한 Radical Candor

### 🚀 상황별 적용

**개발팀과의 협업**
```
Care: "기술 부채 해결이 중요한 거 이해해요"
Challenge: "하지만 이번 스프린트 목표도 달성해야 해요"
Balance: "어떻게 둘 다 진전시킬 수 있을까요?"
```

**디자이너와의 협업**
```
Care: "디자인 퀄리티에 대한 열정 존경합니다"
Challenge: "출시 일정도 중요한 제약이에요"
Balance: "MVP 범위를 함께 정의해볼까요?"
```

**경영진 보고**
```
Care: "비즈니스 압박 이해합니다"
Challenge: "현실적인 타임라인이 필요해요"
Balance: "리스크와 대안을 투명하게 공유할게요"
```

### 📊 피드백 주기

**스프린트 레트로**
- 팀 전체 피드백
- 프로세스 개선
- 성과 축하

**1-on-1**
- 개인 성장 피드백
- 커리어 대화
- 관계 구축

**실시간**
- 즉각적 인정
- 빠른 코스 수정
- 작은 개선

## 흔한 실수와 극복

### ⚠️ 함정들

**Culture Fit 핑계**
- "우리 문화가 그래"
- "한국에선 어려워"
→ 작은 것부터 시작

**Feedback Sandwich 🥪**
- 칭찬-비판-칭찬
- 메시지 희석
→ 명확하고 분리해서

**Personality 핑계**
- "나는 원래 직설적이야"
- "나는 소심해서"
→ 스킬로 접근

### 🎯 개선 전략

**Practice with Low Stakes**
- 작은 일부터 연습
- 안전한 환경에서
- 점진적 확대

**Ask for Feedback**
- "제 피드백이 도움 되었나요?"
- "어떻게 전달하면 좋을까요?"
- 상호 피드백 문화

**Model the Behavior**
- 먼저 보여주기
- 피드백 받는 모습
- 취약성 보이기

## 측정과 개선

### 📈 효과 측정

**팀 지표**
- 심리적 안전감 서베이
- 피드백 빈도
- 1-on-1 만족도
- 팀 성과

**개인 지표**
- 360 피드백
- 성장 속도
- 관계 품질
- 영향력

### 🔄 지속적 개선

**Reflection Questions**
- 오늘 누구에게 피드백 줬는가?
- Care와 Challenge 균형은?
- 받은 피드백은 무엇인가?
- 다르게 할 수 있었던 것은?

## 실천 도구

### 📝 템플릿

**1-on-1 Agenda**
```
1. Check-in (5분)
   - 기분/에너지 레벨
   - 주요 이슈

2. Progress (15분)
   - 지난 주 액션 아이템
   - 현재 진행 상황
   - 블로커

3. Feedback (15분)
   - Praise
   - Coaching
   - Evaluation

4. Growth (10분)
   - 학습 목표
   - 커리어 논의
   - 스킬 개발

5. Action Items (5분)
   - 다음 스텝
   - 약속 정리
```
