UX/UI 디자인(2)
인지과학과 디자인 심리학

사전 질문(15분)
최근에 사용한 앱이나 웹사이트에서 '이거 불편한데 왜 계속 쓰고 있지?'라고 생각한 경험이 있나요? 계속 사용하게 된 이유는 무엇이었나요?

처음 사용했을 때 '정말 잘 만들었다'라고 감탄했던 앱이나 서비스가 있나요? 그런 인상을 만든 구체적인 요소는 무엇이었나요?

사전 질문 토론
1. 최근 사용하면서 불편하지만 계속 쓰는 앱/웹사이트와 이유
대체제가 없거나 필수 기능 때문에 계속 사용 중인 앱(고용24, PASS, 애플 키노트 등).
불편함에도 사용 이유는 신뢰성, 일상화, 혹은 가격 등의 이유 때문.
출석 관리, 본인 인증, 비용 효율 등 서비스의 필요성이 지속 사용의 이유가 됨.
사용성 불편을 보완할 수 있는 대안 부재가 사용자 행동에 큰 영향.
2. 처음 사용했을 때 ‘잘 만들었다’고 감탄한 앱/서비스와 이유
카카오뱅크, 쿠팡, 배민 등 UI/UX가 직관적이고 사용자 중심 설계가 뛰어남.
필요 기능이 명확히 배치되고 사용 흐름이 단순하며 즐거움까지 더함.
UX 라이팅, 친숙한 아이콘, 유도성 높은 디자인 등도 긍정적 인상 제공.
사용자 입장에서 불편함을 최소화하고 효용성을 높인 점이 인상적임.

Case: 토끼 얼굴은 당근을 이길 수 있는가?
앱 아이콘을 당근에서 토끼 얼굴로 바꾸자는 의견
얼굴을 닮은 아이콘을 사람은 본능적으로 선택한다는 심리학 이론
앱 아이콘을 바꿨을 때 다운로드 전환율이 어떻게 되는지 A/B 테스트

결과: 전환율이 30% 이상 증가
별것 아닌 변화이지만 우리의 인지에 영향을 주는 일들이 정말 많다
UX 설계에서는 이런 인지 하나하나가 디자인에 큰 영향을 준다

디자인은 인간의 마음을 다루는 일
"디자이너는 인간 행동의 관찰자이자 예측자여야 한다"
디자인 실패의 90%는 인간에 대한 이해 부족에서 온다
기술적으로 완벽하지만 사용하기 어려운 제품
논리적으로 타당하지만 직관적이지 않은 인터페이스
많은 기능을 제공하지만 원하는 것을 찾을 수 없는 서비스
"인간의 마음은 어떻게 작동하며, 이를 디자인에 어떻게 적용할 것인가?"

인간은 어떻게 생각하는가
System 1과 System 2: 두 가지 사고 모드
우리의 뇌는 두 가지 완전히 다른 방식으로 작동합니다.
System 1: 빠른 사고 (Fast Thinking)
자동적, 직관적, 무의식적
노력이 거의 들지 않음
동시에 여러 작업 가능
진화적으로 오래된 시스템
System 2: 느린 사고 (Slow Thinking)
의식적, 논리적, 분석적
많은 에너지와 집중력 필요
한 번에 한 가지만 처리
상대적으로 최근에 진화

두 시스템의 작동 방식
System 1이 작동하는 순간:
얼굴 표정을 보고 감정 파악하기
"2 + 2 = ?" 의 답 알기
큰 소리가 난 방향 쳐다보기
자전거 타기 (숙련된 후)
System 2가 필요한 순간:
"17 × 24 = ?" 계산하기
복잡한 계약서 검토하기
시끄러운 곳에서 특정 목소리에 집중하기
새로운 소프트웨어 배우기
핵심 통찰: 사용자는 항상 System 1을 선호한다. System 2는 피곤하기 때문이다.

디자인 시사점: 인지 부하를 줄여라
System 1 친화적 디자인의 특징
즉각적 인식
아이콘이 무엇을 의미하는지 바로 이해
버튼인지 아닌지 고민할 필요 없음
다음 단계가 명확함
패턴 인식 활용
익숙한 레이아웃 사용
일관된 상호작용 패턴
예측 가능한 결과
최소한의 의사결정
선택지를 제한
기본값 제공
명확한 우선순위

이중 부호화 이론: 시각과 언어의 협력
인간의 뇌는 시각 정보와 언어 정보를 별개의 채널로 처리한다 - Allan Paivio (1971)
두 가지 독립적 처리 시스템
시각 처리 시스템 (Imagery System)
그림, 도형, 색상, 공간 관계
동시적, 전체적 처리
빠른 인식, 감정적 반응
언어 처리 시스템 (Verbal System)
텍스트, 음성, 숫자, 논리
순차적, 선형적 처리
정확한 이해, 논리적 사고
핵심 원리: 이중 강화 효과
하나의 채널만 사용 = 50% 기억률
두 채널 동시 사용 = 80%+ 기억률
상호 보완적 작용 = 이해도 극대화

변화 거부감 (Change Aversion)
"인간은 현재 상태를 유지하려는 강력한 편향을 가진다"
변화의 이익이 손실보다 2-3배 커야 행동한다
변화를 거부하는 심리적 이유
1. 인지적 노력 회피
새로운 것을 배우는 데 에너지 소비
System 2 사용 강요 = 피로감
"잘 쓰고 있는데 왜 바꿔?"
2. 통제감 상실에 대한 두려움
익숙한 것 = 예측 가능 = 안전
새로운 것 = 불확실성 = 위험
실수할 가능성에 대한 불안
3. 매몰 비용의 오류
기존 방식에 투자한 시간과 노력
"이제 겨우 익숙해졌는데..."

변화 거부감 극복 전략
1. 옵트인(Opt-in) 방식
"새로운 버전 체험해보기"
언제든 돌아갈 수 있음 강조
선택권 제공 = 통제감 유지
2. 브릿지(Bridge) 제공
구버전 요소 일부 유지
친숙한 것에서 시작
단계적 변화 도입
3. 가치 먼저 보여주기
변화의 이유와 혜택 명확히
작은 성공 경험 제공
"이전보다 30% 빨라졌습니다"

변화 거부감 실전 사례 - Gmail의 새 디자인 도입 전략
실패한 접근:
하룻밤 사이 전면 개편
사용자 반발과 혼란
"원래대로 돌려놔!"
성공한 접근:
선택적 체험: "새 Gmail 사용해보기"
되돌리기 옵션: 언제든 클래식 버전으로
단계적 마이그레이션:
1단계: 비주얼만 변경
2단계: 새 기능 추가
3단계: 구버전 단계적 종료
혜택 강조: "더 빠른 검색", "더 큰 용량"
결과: 자연스러운 전환, 저항 최소화

헤일로 효과 (The Halo Effect)
"하나의 긍정적 특성이 전체 평가를 지배한다"
첫인상이 만드는 인지적 후광
작동 메커니즘
인지적 일관성 추구
뇌는 모순을 싫어한다
첫 인상과 일치하는 정보 선호
반대 증거는 무시하거나 합리화
실험 증거:
매력적인 사람 = 더 유능해 보임
유명 브랜드 제품 = 품질도 좋을 것
깔끔한 디자인 = 기능도 우수할 것

헤일로 효과의 양면성
긍정적 헤일로
첫인상이 좋을 때:
작은 결함도 너그럽게 용서
긍정적 기대감 형성
브랜드 충성도 증가
디자인 적용:
온보딩 첫 화면 최적화
로딩 화면도 브랜드 경험
마이크로인터랙션 디테일
부정적 헤일로 (Horn Effect)
첫인상이 나쁠 때:
좋은 기능도 과소평가
부정적 확증 편향
이탈률 증가
위험 신호:
느린 첫 로딩
복잡한 회원가입
오류로 시작하는 경험

선택적 주의와 콘텐츠 맹시
"우리는 보고자 하는 것만 보고, 나머지는 존재하지 않는 것처럼 무시한다"
선택적 주의의 메커니즘
주의력은 제한된 자원
동시 처리 가능 정보: 극히 제한적
생존을 위한 필터링 시스템
관련 없는 정보는 자동 차단
실험적 증거:
농구공 패스 세기 → 고릴라 못 봄 (50%)
배너 광고 클릭률 → 0.05% (배너 맹시)
중요 공지도 놓침 → 목표와 무관하면

콘텐츠 맹시의 패턴들
1. 배너 맹시 (Banner Blindness)
무시되는 영역:
페이지 상단 배너
우측 사이드바
광고처럼 보이는 모든 것
원인:
수년간 학습된 광고 회피
F-패턴 읽기 습관
목표 지향적 탐색
2. 변화 맹시 (Change Blindness)
놓치는 변화들:
점진적 UI 변경
주변부 업데이트
애니메이션 없는 상태 변화

주의력 확보 전략
1. 패턴 깨기 (Pattern Breaking)
효과적인 방법:
예상 위치에서 벗어나기
콘텐츠 중간에 중요 정보 배치
일반적이지 않은 형태 사용
주의사항:
너무 자주 사용하면 새로운 맹시 생성
사용성 해치지 않는 선에서

주의력 확보 전략
2. 맥락적 배치 (Contextual Placement)
사용자 목표와 일치시키기:
결제 단계 → 배송 정보
에러 발생 → 해결 방법
작업 완료 → 다음 단계
타이밍이 핵심:
필요한 순간에 제공
목표 달성 경로상에 배치

인지 편향 - 뇌가 만드는 착각들
편향(Bias)은 버그가 아니라 기능이다
인지 편향은 진화 과정에서 생존을 위해 발달한 mental shortcuts입니다.
빠른 의사결정을 가능하게 함
에너지를 절약함
대부분의 상황에서 "충분히 좋은" 결과 제공
하지만 디지털 환경에서는 이것이 오작동할 수 있다

양날의 검: 편향의 이중성
사용자의 편향
제품을 어떻게 인식하는가
어떤 기능을 선택하는가
무엇을 기억하고 평가하는가
디자이너의 편향
리서치를 어떻게 해석하는가
솔루션을 어떻게 선택하는가
결과를 어떻게 평가하는가
"당신이 편향에서 자유롭다고 생각한다면, 그것이 바로 편향 맹점(Bias Blind Spot)이다"

1. 앵커링 효과 (Anchoring Bias)
원리: 처음 접한 정보가 이후 판단의 기준이 된다
실제 사례:
가격 표시: ₩99,000 → ₩59,000 (40% 할인!)
요금제 선택: 프리미엄을 먼저 보여주고 베이직 제시
진도 표시: "이미 70% 완료했습니다!"
디자인 적용:
비교의 기준점을 전략적으로 설정
첫인상이 중요한 온보딩 경험 설계
기대치를 관리하는 커뮤니케이션

2. 확증 편향 (Confirmation Bias)
원리: 기존 믿음을 지지하는 정보만 선택적으로 받아들인다
실제 사례:
넷플릭스 추천: "당신이 좋아할 만한 콘텐츠"
아마존: "이 상품을 본 고객이 함께 본 상품"
소셜미디어 필터 버블
디자인 적용:
개인화는 하되, 다양성도 노출
사용자의 기대와 일치하는 피드백 제공
반대 의견이나 대안도 부드럽게 제시

3. 손실 회피 (Loss Aversion)
원리: 잃는 것에 대한 고통이 얻는 것의 기쁨보다 약 2배 크다
실제 사례:
"마지막 3개 남음!" (품절 임박 알림)
"프리미엄 혜택을 잃고 싶지 않으시다면..."
"작성 중인 내용이 사라집니다"
디자인 적용:
진행 상황 자동 저장 강조
이탈 시 잃게 될 것 명시
한정 수량/시간 전략적 활용

4. 사회적 증거 (Social Proof)
원리: 불확실한 상황에서 다른 사람의 행동을 따라한다
실제 사례:
"1,000명이 지금 이 상품을 보고 있습니다"
별점과 리뷰 시스템
"베스트셀러" 뱃지
디자인 적용:
사용자 수, 리뷰 수 표시
실시간 활동 표시
커뮤니티 피드백 가시화

5. 더닝-크루거 효과
"아는 것이 적을수록 자신감은 넘친다"
자신감의 역설
초보자: 과도한 자신감 (무지의 정점)
중급자: 자신감 하락 (절망의 계곡)
전문가: 적절한 자신감 (지혜의 고원)
온보딩 설계
초보자의 과신 활용: "당신도 할 수 있습니다!"
단계적 난이도 상승
초기 성공 경험 제공
예시: 듀오링고
첫 레슨은 극도로 쉽게
"나도 언어를 배울 수 있구나!"
점진적으로 복잡도 증가

6. 매몰비용 오류 (Sunk Cost Fallacy)
"이미 투자했으니 계속해야 해"
사용자 측면:
게임: "레벨 50까지 키웠는데..."
구독: "6개월 썼는데 이제 와서...
디자이너 측면:
"이 기능에 3개월 투자했는데..."
데이터가 부정적이어도 고집

7. 밴드왜건 효과 (Bandwagon Effect)

14. 공통 운명의 법칙 (Law of Common Fate)

15. 목표 기울기 효과 (Goal Gradient Effect)
목표에 가까워질수록 동기부여가 증가한다
완료가 가까이 보일 때 노력이 가속화된다
실험적 증거:
커피 스탬프 카드 연구
10개 중 2개 채워진 카드 > 8개 빈 카드
실제로는 같은 8개지만 완성률 차이
심리적 원리:
진전 가시화가 핵심
끝이 보이면 포기율 감소
작은 성취가 큰 동력

16. 피크엔드 법칙 (Peak-End Rule)
경험의 기억 = (절정의 순간 + 마지막 순간) ÷ 2
전체 경험보다 가장 강렬한 순간과 마지막이 중요하다
실제 사례:
병원 대장내시경 연구
A그룹: 짧지만 고통스러운 끝
B그룹: 더 길지만 편안한 끝
결과: B그룹이 경험을 더 긍정적으로 기억
디자인 적용:
Peak: 목표 달성의 순간을 축하
End: 긍정적인 마무리 경험 설계
