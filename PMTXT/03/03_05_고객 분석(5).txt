고객 분석(5)
데이터에서 행동을 발견하기

사전 질문(15분)
기획한 의도대로 고객이 제품을 사용하지 않는다면 그 이유는 무엇일까?

제품에 고객이 오랫동안 머물게 하기 위해서 어떤 전략이 필요할까?

사전 질문 토론
✅ 기획한 의도대로 고객이 제품을 사용하지 않는 이유  
1. 고객의 니즈와 불일치 – 기획자가 해결하려던 문제와 고객이 원하는 해결책이 다를 수 있음.  
2. UI/UX 문제 – 사용성이 직관적이지 않거나, 제품의 가치가 제대로 전달되지 않음.  
3. 고객의 새로운 활용 방식 – 의도와 다르게 고객이 더 적합한 용도를 발견하여 사용함.  

✅ 제품에 고객이 오랫동안 머물게 하는 전략  
1. 지속적인 피드백 반영 – 데이터를 기반으로 업데이트하며 고객 경험을 개선.  
2. 직관적인 UI/UX 설계 – 사용하기 쉽고, 자연스럽게 제품을 이해하도록 유도.  
3. 보상 및 커뮤니티 요소 활용 – 포인트, 게이미피케이션, 커뮤니티 형성을 통해 참여를 지속적으로 유도.

Case: 온보딩이 길어졌는데 전환율이 늘었다?
경쟁 제품을 분석하던 중 특이한 패턴을 발견
우리 제품보다 가입에 필요한 시간이 3배 이상 길었는데 매출은 훨씬 높았다
두 제품의 차이
우리 제품: 가입 절차를 간소화해서 빠르게 가입하도록 유도
경쟁 제품: 고객에게 제품과 관련된 여러가지 질문들을 수십가지 물어봄

경쟁 제품의 전환율이 더 높았던 이유는 무엇일까?

긴 온보딩이 고객의 문제를 해결해 주었다
짧은 온보딩: 제품이 필요해서 가입했지만 가입 이후에 제품을 사용하기 전까지는 이 제품이 유용한지 느낄 수 없었다
긴 온보딩: 여러 질문에 답하는 과정에서 이 제품이 나에게 도움이 될것이라는 기대감을 형성하였고 오히려 가입 및 구매 전환을 더 쉽게 만들었다

정성 분석으로 발견한 고객 가치를 온보딩의 개선을 통해 가입 과정에서 고객에게 전달
경쟁 제품과 마찬가지로 가입 전환율 상승!

정량 분석은 ‘고객 가치'가 동작하는지 이해하는 도구
고객 기회 발견의 핵심은 ‘행동'과 ‘감정'의 이해
정성적 분석을 통해 그 가치가 동작하는 ‘방식'을 이해했다면
정량적 분석은 그 가치가 실제로 동작하는지 ‘검증'을 하는 도구

고객이 어떤 문제를 해결하고 싶은지 정성 분석을 통해 발견한다면
그 문제가 실제로 해결되고 있는지는 정량 분석을 통해 발견한다

정량 분석 = 행동 데이터 분석
우리가 제품을 통해 구현해내야 하는 것
문제 해결에 대한 기대 -> 행동 실행 -> 실제로 문제가 해결
예) 셀프 케어 서비스
자신의 멘탈에 도움이 될것을 기대 -> 기능 실행 -> 실제 나의 멘탈 문제가 해결

모든 정량 분석은 고객이 문제해결의 ‘행동'을 실제로 수행했는지를 확인하는 과정

제품 기획에서 행동 데이터가 중요한 이유

제품을 만들어도 고객이 사용하지 않는다면 그것은 문제가 해결되지 않았다는 뜻
고객이 그 기능을 사용하지 않는 이유
그 기능이 문제를 해결할 것이라는 ‘기대'가 생기지 않아서
그 기능을 실제로 ‘행동’하는데 어려움이 있어서
그 기능을 수행했지만 실제로 문제가 ‘해결’되지 않아서

“아무리 좋은 기능이라도 고객이 사용하지 않으면 고객의 문제를 해결해 주지 못한다"

PM이 데이터를 바라보는 입장


PM은 고객이 해결하고자 하는 문제의 주인이 되어야 한다
데이터는 이 제품이 고객 문제를 해결해 주는지 발견하는 실마리가 된다
PM은 비즈니스가 성장하는 문제의 주인이 되어야 한다
데이터는 이 제품이 비즈니스 성과가 나고 있는지를 확인할 수 있게 한다
PM은 제품팀이 우선순위를 선정하는 문제의 주인이 되어야 한다
데이터는 제품팀이 최우선적으로 해결해야 하는 문제를 설득할 근거가 된다

행동 데이터에서 패턴 찾기


핵심 고객 행동 분석
고객이 우리 제품에서 ‘가치'를 이해하고 이에 맞게 사용하는 행동

비정상적인 행동 분석
고객이 우리 제품에서 ‘가치'를 이해하지 못하고 잘못 사용하는 행동

핵심 고객 행동(1)
활성 사용자 행동 (Engagement Behavior)
DAU/WAU/MAU (일간/주간/월간 활성 사용자)
활성 유저가 특정 핵심 기능을 얼마나 자주 사용하는지
연속적인 사용 패턴 (예: "지난 7일간 3일 이상 접속한 비율")

핵심 고객 행동(2)
반복 행동 및 습관 형성 (Retention & Stickiness)
사용 빈도 분석 (Frequency of Use) → "일주일에 몇 번 사용하는가?"
세션 지속 시간 (Session Duration) → "한 번 사용할 때 얼마나 오래 사용하는가?"
습관적 사용 패턴 → "매일 특정 시간대에 사용하는가?"

핵심 고객 행동(3)
제품 핵심 가치 경험 (Core Feature Usage)
고객이 제품의 핵심 기능을 경험했는지 분석
예) 음악 스트리밍 서비스 → "첫 10곡을 끝까지 들은 비율"
예) 이커머스 → "첫 구매까지 걸린 시간"

핵심 고객 행동(4)
추천 및 네트워크 효과 (Referral & Virality)
사용자가 다른 사람을 초대하는가? (초대 링크 클릭, 공유율 등)
커뮤니티 제품이라면, 사용자가 얼마나 자주 댓글/좋아요를 남기는지
PM이 네트워크 효과가 발생하는 주요 지점을 파악하는 법

추천 및 네트워크 효과가 중요한 이유
제품이 가치가 있으면 누군가에게 이를 공유하고 싶어하는 마음
생존 본능으로 인해 나와 같은 문제를 겪는 사람에게 이 제품을 공유하고 싶다
한 사람이 평균 1.5명에게 추천을 해도: 10단계를 거치면 58배 상승

“ 제품의 성장을 위해서 공유 & 추천 기능으로 네트워크 효과를 만드는 것은 선택이 아닌 ‘필수'이다"

비정상적인 행동(1)
이탈 행동 (Churn Behavior)
즉각적인 이탈 (Immediate Churn) → 가입 후 몇 분/몇 시간 내에 이탈하는 비율
점진적인 이탈 (Gradual Churn) → 주간/월간 활성 사용자 감소 패턴 분석
이탈 유저와 유지 유저의 행동 비교 → "이탈한 유저가 특정 기능을 전혀 사용하지 않았는가?"

비정상적인 행동(2)
기대 대비 비정상적 행동 (Unexpected Behavior)
기능 사용 과정에서 "기대와 다른 방식"으로 사용한 경우
예) 검색 기능을 이용해야 하는데 홈 화면을 스크롤해서 원하는 제품을 찾음 → 검색 기능 UI 개선 필요
예) 예약 기능을 제공하지만, 대부분의 사용자가 고객센터에 직접 문의함 → 예약 기능 신뢰 문제?

비정상적인 행동(3)
제품 사용 오류 및 문제 행동 (Error & Misuse)
반복적인 오류 발생 (버튼 클릭 후 아무 반응 없음 등)
예상하지 못한 사용자 행동 (예: 가입 후 같은 페이지에서 계속 머무름, 특정 페이지에서 이탈 급증 등)

비정상적인 행동(4)
이상치 탐지 (Anomaly Detection)
평균적인 사용 패턴과 큰 차이를 보이는 유저 그룹 발견
예: 대부분의 고객이 5분 동안 사용하는데, 특정 그룹이 30초 만에 이탈하는 경우

"이탈 유저"가 보이는 특징
📌 이탈 고객 분석이 중요한 이유
신규 유저의 초반 이탈을 줄이면 장기적인 리텐션이 개선됨.
고객 획득 비용이 높은 만큼, 이탈 방지 전략이 더 중요 (CAC vs. LTV 관점)

"이탈 유저"가 보이는 특징
즉각적인 이탈 (Immediate Churn) - 가입 후 1~3일 내에 이탈하는 고객
가입 후 핵심 가치를 경험하기 전 이탈
기대한 기능이 보이지 않음
제품 사용법이 어려움

온보딩 과정에서 "가입을 완료한 유저" vs. "완료하지 않은 유저" 비교 → 이탈 차이 확인
솔루션: 온보딩 과정 최적화, 첫 경험 개선

"이탈 유저"가 보이는 특징

(2) 점진적인 이탈 (Gradual Churn) - 한 달 동안 점차적으로 방문 빈도가 줄어드는 유저
제품이 반복 사용할 필요성이 낮음
경쟁 제품과 비교했을 때 차별성이 부족
"이벤트 트리거"가 부족하여 재방문할 이유가 없음

이커머스 앱 → 구매 후 앱을 다시 열지 않는 고객 패턴
생산성 앱 → 첫 사용 후 7일간 방문이 없는 유저의 특징 분석
솔루션: 리마인드 푸시, 개인화 추천, 재방문 유도

"이탈 유저"가 보이는 특징

이탈 유저의 분석이 어려운 이유
한번 떠난 고객은 이유를 남기지 않는다
무언가를 쓰지 ‘않는’ 이유를 알아내는 것은 어려운 일이다

“고객 ‘이탈'을 줄이지 않으면 고객을 ‘많이' 모으는 것의 의미가 사라진다"

"충성 고객"이 보이는 특징


📌 충성 고객을 분석하는 이유
충성 고객(High Retention User)이 보이는 행동을 이해하면, 그 패턴을 신규 유저에게 확산할 수 있음
마케팅/온보딩 최적화를 위해 "충성 유저의 첫 경험"을 분석해야 함

"충성 고객"이 보이는 특징


(1) 충성 고객이 반복적으로 수행하는 핵심 행동
사용 빈도(Frequency of Use): 주간/월간 반복 사용 패턴
습관적 행동 (Habit Formation): 특정 요일, 시간대에 반복 사용
기능 집중도 (Feature Engagement): 특정 기능 사용 비율이 높은지

SNS 앱 → 1주일에 3번 이상 게시물 작성하는 고객 = 충성 고객
음악 앱 → 특정 플레이리스트를 저장하는 행동이 유지율과 연결됨
생산성 앱 → 매일 특정 기능(메모, 태스크 정리) 사용 = 리텐션 높은 고객

"충성 고객"이 보이는 특징


(2) 충성 고객이 초반에 보인 행동 패턴 (First 7 Days Rule)
가입 후 7일 내에 특정 행동을 한 유저 vs. 하지 않은 유저의 비교 분석
충성 고객이 가입 후 처음 7일 동안 어떤 행동을 했는지를 찾으면, 신규 유저도 같은 경험을 하도록 유도 가능

이커머스 → "첫 구매까지 걸린 시간"이 짧은 유저가 리텐션이 높음
SNS → "첫 3일간 5명 이상 친구 추가한 유저"가 장기적으로 남아 있음
솔루션: 초기 경험 개선, 유도 트리거 설정 (예: 첫 구매 유도 할인, 친구 추천 보너스 등)

신규 고객 vs 기존 고객의 행동 차이 분석

📌 신규 vs 기존 고객 분석이 중요한 이유
신규 고객과 기존 고객이 제품을 사용하는 방식이 다름
이 차이를 이해하면, 신규 유저가 기존 유저처럼 사용할 수 있도록 유도할 수 있음

신규 고객 vs 기존 고객의 행동 차이 분석

(1) 제품 탐색 방식이 다름
신규 유저: 메인 화면을 탐색하는 비율 높음, 메뉴 이동 빈번
기존 유저: 바로 특정 기능/페이지로 이동하는 비율 높음

이커머스 → 신규 유저는 "상품 목록 스크롤"이 많고, 기존 유저는 "검색"을 더 많이 사용
음악 앱 → 신규 유저는 "추천 곡"을 더 많이 듣고, 기존 유저는 "자신의 플레이리스트"를 더 활용
솔루션: 신규 유저를 위한 맞춤형 온보딩 UI 제공

신규 고객 vs 기존 고객의 행동 차이 분석

(2) 특정 기능의 사용률 차이
기존 유저는 커스텀 기능(설정 변경, 즐겨찾기 추가 등) 활용도가 높음
신규 유저는 기본 기능(검색, 추천) 위주로 사용

Notion → 신규 유저는 템플릿을 활용하지만, 기존 유저는 빈 페이지에서 시작하는 경우 많음
솔루션: 신규 유저를 위한 "템플릿 추천" 기능 제공하여 기존 유저처럼 빠르게 적응하도록 유도

Break Time

제품 가치를 ‘사용'하는 것을 넘어 ‘습관'이 되게 하려면
어떤 제품은 몇번 사용하다 잊혀지지만 어떤 제품은 끊임없이 접속한다.
사용자가 애착을 갖고 중독되어 사용하게 되는 제품들이 있다

“습관처럼 사용하는 상품은 어떻게 만들어 지는가?” 

훅(Hooked)

Hook 모델이란
사용자의 반복 행동을 유도하는 4단계 구조 
Trigger → Action → Reward → Investment
사용자가 특정 제품을 습관적으로 사용하게 만드는 원리
Facebook, Instagram, TikTok 등 글로벌 플랫폼이 Hook 모델을 활용한 사례

“잘나가는 서비스들은 거의 ‘모두'가 중독을 모델링해 냈다”

Hook 모델이 동작하면 어떤 효과를 내는가
1) 리텐션 증가 → 지속적인 사용자 확보
Hook 모델이 동작하면, 사용자 유지율(리텐션)이 크게 향상됨.
예: TikTok의 알고리즘 → "다음 영상이 궁금해서 계속 사용"
2) 유저 이탈 방지 → CAC 대비 LTV 증가
신규 고객을 유치하는 비용(CAC)이 높아지는 상황에서, Hook 모델이 유저를 유지시키는 역할.
고객이 쉽게 이탈하지 않도록 반복 사용을 유도함.
3) 제품 성장 → 네트워크 효과 촉진
사용자들이 제품을 자발적으로 확산시키는 과정이 Hook 모델과 연결됨.
예: Instagram에서 "좋아요"와 "댓글"이 재방문을 유도하는 방식.


Trigger – 행동을 시작하게 만드는 요소
외부 트리거 (External Trigger) vs. 내부 트리거 (Internal Trigger)

✅ 외부 트리거: 제품 외부에서 행동을 유도하는 요소
“What does the user really want?”
예: 푸시 알림, 이메일, 배너 광고, 친구 초대 메시지
사례: "Instagram이 좋아요/댓글 알림을 활용하는 방식"

Trigger – 행동을 시작하게 만드는 요소
✅ 내부 트리거: 사용자의 감정, 습관, 경험에서 발생하는 행동 유도 요소
“What get the user to the product?”
예: 외로울 때 SNS를 켜는 행동, 무료할 때 쇼핑 앱을 여는 행동
사례: "TikTok이 무료함(내부 트리거)과 연결되는 방식"

Action – 사용자가 행동을 수행하는 과정
✅ BJ Fogg의 행동 모델 (B=MAT)과 연결
M: 동기(Motivation) 높을수록 행동 가능성이 증가
A: 행동이 쉬울수록 실행 확률 증가
T: 트리거가 적절한 순간 제공되면 행동 유도 가능
📌 트리거를 설계할 때 고려해야 할 점
외부 트리거가 반복되면 내부 트리거로 전환되는 원리는?
"어떤 감정 상태에서 우리 제품이 떠오르게 만들 것인가?"

Action – 사용자가 행동을 수행하는 과정
✅ 사용자의 행동을 단순화하는 방법
클릭 수 줄이기, UI 간결화, "가장 쉽게 행동하도록 만드는 UX 설계"
사례: "Amazon의 1-Click 결제 → 사용자의 구매 행동을 최소한의 노력으로 연결"

"우리 제품의 핵심 행동(Action)은 무엇인가?"
"이 행동을 수행하기 어렵게 만드는 요소는 없는가?"

Reward – 보상을 통한 행동 강화
📌 "보상(Reward)이 반복 행동을 유도하는 핵심 요소"
✅ 보상의 3가지 유형 (Nir Eyal의 Hook 모델 원칙 적용)
사회적 보상 (Social Reward): 좋아요, 댓글, 공유, 팔로워 증가
자기 만족형 보상 (Self-Reward): 학습 진도율, 성취감, 목표 달성
변동 보상 (Variable Reward): 랜덤 보상, 예상할 수 없는 이벤트

Reward – 보상을 통한 행동 강화
✅ 보상의 불확실성이 행동을 반복하게 만드는 원리
"도파민 시스템과 보상의 기대 효과"
사례: "TikTok의 알고리즘이 사용자에게 가변적 보상을 제공하는 방식"
📌 Reward 설계할 때 고려해야 할 점
보상이 너무 자주 주어지면 무뎌질 수 있음 → 적절한 간격 유지
기대하지 않은 보상이 더 강력한 효과를 가짐 → "서프라이즈 효과 활용"

Investment – 미래 행동을 유도하는 요소

📌 Investment가 중요한 이유
사용자가 제품에 시간, 데이터, 감정, 돈을 투자할수록 이탈 확률 감소
반복적인 투자로 제품과 정서적 연결 형성

Investment – 미래 행동을 유도하는 요소

✅ Investment의 주요 유형
데이터 입력 (Personalization) → 사용자가 입력한 데이터가 많을수록 제품을 떠나기 어려움
소셜 그래프 연결 (Friend & Network) → 친구를 추가할수록 제품 이탈 가능성이 낮아짐
콘텐츠 생성 (User-Generated Content) → 작성한 콘텐츠가 많을수록 이탈 어려움

Investment – 미래 행동을 유도하는 요소

📌 Investment 설계할 때 고려해야 할 점
"어떤 투자 행동이 유저를 더 강하게 묶어놓을 것인가?"
"이 투자가 지속적으로 더 많은 트리거를 생성하게 만들 수 있는가?"

Hook 모델 실무 적용 사례(1)

✅ 1. Instagram – 습관 형성 제품의 대표 사례
트리거: 친구의 사진 업로드 푸시 알림
액션: 앱 실행 후 사진 확인
보상: 좋아요/댓글을 통한 피드백
투자: 팔로워, 사진 업로드 → 계정 활성화 유지

Hook 모델 실무 적용 사례(1)

✅ 2. TikTok – 가변적 보상을 활용한 강력한 리텐션 모델
트리거: 푸시 알림, 추천 동영상
액션: 영상 시청 (스크롤 한 번으로 쉽게 가능)
보상: 예측 불가능한 재미있는 영상 (가변적 보상)
투자: 좋아요, 댓글, 팔로우로 네트워크 효과 증가

Hook 모델 실무 적용 사례(3)

✅ 3. Notion – 생산성 도구에서 Hook 모델이 적용된 사례
트리거: 메모, 문서 정리 필요성
액션: 문서 생성 후 콘텐츠 작성
보상: 정리된 문서를 통해 생산성 향상 경험
투자: 문서와 데이터가 많아질수록 서비스 이탈 어려움

행동 데이터와 Hook 모델의 연결
Hook 모델이 잘 동작하고 있다면? 리텐션(재방문율)이 증가한다
설계된 Hook 모델의 각 행동에 해당하는 지표를 지속적으로 모니터링 한다

무엇에 중독되게 할 것인가?
훅 모델을 만들어 낸다고 억지로 중독되지는 않는다
제품이 제공하는 핵심 가치에 중독되게 하는 것
제품 가치와 고객 기회를 이해하고 있지 않으면 훅 모델은 무의미 하다
고객의 ‘데이터'에만 집중하지 않고 고객에게 제공하는 ‘가치'가 우선되어야 한다

리텐션을 올리는 기술같은 것은 존재하지 않는다.
제공하고자 하는 가치를 더욱 다듬는 것이
제품에 대한 애착을 만든다.
