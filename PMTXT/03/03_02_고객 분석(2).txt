사전 질문 토론
고객의 문제를 발견하려면 어디서 정보를 얻어야 할까?
직접 조사: 인터뷰, 설문조사, 현장 리서치 등을 통해 고객의 실제 경험과 의견을 파악.
데이터 분석: 클릭률, 이탈률, 사용 패턴 등의 행동 데이터를 기반으로 문제 발견.
경쟁사 및 시장 조사: 경쟁사 서비스 분석, 업계 전문가 인터뷰 활용.
사용자 피드백: SNS, 리뷰, Q&A 등에서 직접적인 문제점을 확인.
모든 구성원이 고객 문제를 이해하려면 어떻게 해야 할까?
공통 목표 설정: 고객 문제 해결을 팀의 핵심 미션으로 정의하고 공유.
데이터 시각화: 대시보드, 리포트 등을 통해 문제를 객관적으로 전달.
직접 체험: 팀원들이 서비스 사용 경험을 통해 고객 입장에서 문제를 느끼도록 유도.
소통과 협업: 자주 논의하고 의견을 나누는 문화를 조성해 문제에 대한 공감 형성.

고객 기회: 가장 많이 해결하려는 문제를 발견하는 것

고객 문제 = 비즈니스 기회
문제가 클수록 → 기회도 크다
많은 사람이 겪는 문제일수록 → 시장이 크다
해결이 어려운 문제일수록 → 경쟁 장벽이 높다
예시:
배달 지연 문제 → 실시간 배달 추적 서비스 기회
복잡한 결제 과정 → 원클릭 결제 솔루션 기회
식당 웨이팅 → 모바일 예약 플랫폼 기회

성공한 서비스는 모두 
해결이 어려운 문제를 발견해 
기회를 성공으로 연결시켰다

2. 고객의 피드백과 후기 분석

자주 언급하는 문제 - 핵심 기회일 가능성

2. 고객의 피드백과 후기 분석
피드백은 주관적일 수 있다. 
여러 다양한 출처에서 나온 피드백을 종합하여 패턴을 찾아야
예) 한 명의 고객이 "이 기능이 너무 복잡하다"라고 말한 것만으로는 그것이 큰 문제인지 알 수 없다. 하지만 여러 명의 고객이 동일한 불만을 표현하면, 그 문제는 제품의 주요 문제일 가능성이 높다.
피드백을 잘 분류하고 분석하는 것이 중요하다

3. 고객 인터뷰

기회 찾기 - 정량적 분석
행동 데이터 분석: 고객의 행동 데이터를 통해 문제 발생 지점을 식별합니다.
세그먼트 분석: 고객을 그룹별로 나누어 행동 패턴을 분석하고, 주요 문제를 파악합니다.
코호트 분석: 동일 시점에 유입된 사용자들의 행동을 추적하여 제품 문제와 개선 기회를 발견하는 방법

1. 행동 데이터 분석
사용자들의 실제 행동 패턴을 데이터로 수집하고 분석
어디서 사용자들이 이탈하는지, 어떤 기능을 많이 사용하는지 파악 가능
사용자 여정의 병목 지점과 문제 영역을 객관적으로 식별할 수 있음

2. 세그먼트 분석
사용자를 다양한 기준으로 그룹화하여 행동 패턴 비교
특정 고객군의 고유한 문제점과 니즈 발견 가능
고객 그룹별 가치 제안을 차별화하고 집중할 세그먼트 결정에 도움

3. 코호트 분석
특정 기간에 유사한 경험을 한 사용자 그룹의 행동 추적
시간에 따른 유지율과 이탈률 패턴으로 문제 지점 식별
제품 변화나 개선이 실제 사용자 행동에 미치는 장기적 영향 파악 가능

데이터와 인사이트의 통합
기회를 더 정확하게 발견하려면 정성적 분석과 정량적 분석을 함께 활용해야
VOC와 데이터는 서로를 검증하고 보완
고객이 말하는 문제가 실제 데이터에서도 나타나는지 확인
데이터에서 발견한 패턴의 이유를 고객 피드백에서 찾을 수 있기도

통합 분석의 예시
이탈 데이터: 결제 단계에서 40% 이탈
VOC 분석: "결제 과정이 너무 복잡하다" 불만 다수
인터뷰: 사용자들이 여러 번의 인증 단계에 좌절감 표현
코호트 분석: 결제 단계 개선 후 신규 사용자 유지율 향상

단편적 데이터나 의견보다 종합적 분석에서 더 정확한 기회 발견할 수 있다
정성적 + 정량적 접근이 함께 뒷받침하는 문제에 집중해야 한다

특정 목표를 달성하기 위해 사용자(Customer) 또는 직원(Employee) 이 제품/서비스와의 경험을 시간순으로 겪어가는 과정.
사용자의 관점에서 단계를 시각화하여 표현.
영화 시나리오에서 주인공의 움직임과 유사


사용자 여정 지도의 목표
사용자의 필요와 동기 이해.
사용자 경험에서의 문제점(Gaps, Pain Points) 탐색.
기회(Opportunities) 찾기.
협업 촉진 (공유된 비전).

구성 요소 (Elements) - 유동적으로 추가 가능
타임라인(Timeline)
단계(Phases) – 경험의 각 주요 단계.
사용자(Persona) – 분석 대상 고객(비교 분석 가능).
시나리오(Scenario) – 특정 경험이나 사용 사례.
단계별 행동(Steps) – 순차적 진행 과정.
채널(Channels) – 다중 채널 여정 시각화 (예: 앱, 웹, 대면 경험 등).

구성 요소 (Elements) - 유동적으로 추가 가능
생각(Thoughts) – 사용자 조사 데이터에서 인용 (예: 비디오 녹화, 인터뷰).
감정적 경험(Emotional Experience) – 만족도 및 감정 변화.
드라마틱 아크(Dramatic Arc) – 몰입도(Engagement Level).
이해관계자(Stakeholders) – 해당 단계에서 책임 있는 사람.
인사이트(Insights) – 문제점, 기회, 위험 요소 등. 예) 리스크 분석, 발생할 수 있는 위협 요소.

사용자 여정(User Journey) vs 사용자 흐름(User Flow)

사용자 여정(User Journey) != 사용자 흐름(User Flow)
여정: 감정, 생각, 경험 포함한 전체적 경험
흐름: 시스템 내 이동 경로와 기능적 단계
여정은 사용자 중심, 흐름은 시스템 중심
사용자 여정은 고객이 목표 달성 과정에서 느끼는 감정과 경험에 초점
사용자 흐름은 앱/웹사이트 내 화면 간 이동 경로와 기능적 단계에 집중
여정 맵은 제품 전략과 혁신 기회 발견에 유용
흐름 다이어그램은 UI/UX 설계와 기술 개발에 직접 활용

생각: "어떤 음식점이 맛있을까?", "리뷰가 좋은 곳은?"
감정: 혼란스러움, 결정 어려움
문제점: 정보 과부하, 선택 기준 불명확
기회: 개인화된 추천, 간소화된 비교 기능

배달 음식 여정 맵 - 단계별 분석 예시 (2)
단계: 배달 기다림
행동: 배달 추적, 다른 일 하면서 대기
생각: "언제 도착할까?", "제대로 오고 있을까?"
감정: 불안, 기대감
문제점: 불확실한 도착 시간, 상태 불투명
기회: 실시간 배달 추적, 정확한 도착 예측

초보자가 흔히 하는 실수 & 해결 방법
사용자의 실제 경험이 아닌 내부 가정(Assumption)으로만 맵을 작성함 → (해결) 최소한 기존 사용자 피드백과 데이터를 참고할 것.
단순히 타임라인을 나열하는 데 집중하고, 감정이나 페인포인트 분석이 부족함 → (해결) 감정 곡선, 페인포인트, 기대 vs. 현실 차이를 명확히 추가할 것.
너무 많은 정보를 담아 가독성이 떨어짐 → (해결) 핵심 단계만 먼저 작성하고 점진적으로 확장할 것.
한 번 만들고 끝내버림 → (해결) 주기적으로 업데이트하고 실제 사용자 테스트를 반영할 것.

사용자 여정 맵의 실제 비즈니스 가치
제품 개발팀에게: 우선순위를 명확히 하고 리소스 할당을 최적화
마케팅팀에게: 고객 니즈와 페인포인트에 맞춘 메시지 개발
경영진에게: 투자 결정을 위한 명확한 근거 제공
고객 지원팀에게: 문제 지점에 대한 이해와 대응 방안 마련

여정 맵에서 기회 발견하기
감정 곡선이 급격히 떨어지는 지점 확인
사용자가 우회 경로를 만드는 단계 분석
경쟁사와 비교했을 때 약점이 드러나는 부분
사용자의 기대와 실제 경험 간 격차 지점

처음부터 완벽한 여정 맵을 만들 필요는 없음
초안(가설 기반) → 사용자 피드백 반영 → 반복 개선하는 방식이 정답.
단순한 형태라도 먼저 만들고, 점진적으로 업데이트하는 것이 가장 효과적.
한 번에 완벽한 맵을 만들려 하지 말고, 빠르게 작성하고 빠르게 개선하는 것이 중요함.

사용자 여정 맵이 도구일 뿐인 이유
고객 여정 맵을 작성하는 것이 중요한 것이 아니다
고객의 제일 중요한 문제가 무엇인지 파악하기 위한 '도구'일 뿐
중요한 것은 고객의 제일 심각한 문제가 어디에서 어떻게 발생하는지를 파악하는 것
기능 중심으로 제품을 정리하지 말고 경험 중심으로 정리하는 것이 필요
실제로 수행하는 행위, 구체적으로 느끼는 어려움
다른 이해관계자와의 협업 도구로써 활용

심지어 더 많은 구성요소를 추가할 수도 있다
스토리보드 (Storyboards): 단계별 사용자 경험을 시각화하여 스토리텔링 효과 강화.
사용자의 목표 (Jobs-To-Be-Done): 사용자가 성취하려는 것 → 원하는 감정 & 인식 → 기대 결과 (Desired Outcomes).
전환 퍼널 분석 (Conversion Funnel Analysis): 사용자가 이탈하는 지점 분석. 각 단계에서 전환율 향상 전략 수립.
보조 캐릭터 (Supporting Characters): 주요 사용자 외에도 전체 경험에 영향을 주는 요소가 존재할 수 있음.
결정적 순간 (Moments of Truth): 브랜드/제품과의 관계를 결정짓는 중요한 상호작용.
핵심 성과 지표 (KPI’s): KPI 포함 시 평가 및 개선 방향 설정이 용이.
장애물 및 문제점 (Barriers/Obstacles): 가장 중요한 페인 포인트(Pain Points) 분석.
사용자의 질문 및 불확실성: 사용자(Persona)가 갖고 있는 주요 질문/불확실성 탐색.

사용자 여정 지도는 사고하는 방식
제품을 기능 단위로 이해하게 되면 고객 기회를 발견하기가 어려워진다.
항상 고객이 자신의 삶 속에서 우리 제품을 어느 순간에 마주하는지를 중심으로 사고해야 한다.
사용자 여정 지도는 이 사고를 시각화해놓은 도구에 불과하다
사용법을 자세하게 익히기 보다는 이런 사고를 통해 어떻게 기회를 발견하고 의사소통 할 수 있는지에 대해서 고민해 봐야
모든 팀 구성원들에게 이것을 기반으로 소통하고 공유할 수 있어야
