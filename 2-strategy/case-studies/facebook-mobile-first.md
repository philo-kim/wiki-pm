# Case Study: Facebook의 Mobile First 전환

> 데스크톱 중심에서 모바일 우선으로 - 생존을 건 전략적 피벗

## 🎯 위기 상황 (2012)

### IPO 직후의 충격
- 상장가: $38 → 3개월 후 $18 (50% 하락)
- 모바일 매출: 전체의 0%
- 모바일 사용자: 전체의 54%
- 투자자 신뢰 붕괴

### 핵심 문제
**모바일 사용자는 늘지만 수익화 실패**
- HTML5 웹앱 성능 문제
- 모바일 광고 솔루션 부재
- 데스크톱 중심 조직 구조
- 모바일 개발 인력 부족

## 📱 전략적 결정

### Mark Zuckerberg의 선언 (2012.09)
"우리의 가장 큰 실수는 HTML5에 너무 많이 베팅한 것이다. 이제 모바일이 우선이다."

### Mobile First 원칙
1. **모든 기능은 모바일부터 설계**
2. **데스크톱은 모바일의 확장**
3. **네이티브 앱 집중**
4. **모바일 광고 혁신**

## 🔄 조직 재편

### 구조 변경
```
Before (2012):
- Desktop Team: 80%
- Mobile Team: 20%
- 부서별 사일로

After (2013):
- Mobile First Teams: 100%
- Platform별 아닌 기능별 조직
- 모바일 전문가 대규모 채용
```

### 인재 확보
- Instagram 인수 ($1B, 2012)
- WhatsApp 인수 ($19B, 2014)
- 모바일 엔지니어 3배 증원

## 💡 제품 혁신

### 네이티브 앱 재개발
**iOS 앱 (2012.08)**
- 속도 2배 향상
- 사용 시간 25% 증가
- 일일 활성 사용자 20% 증가

**Android 앱 (2012.12)**
- 완전 재작성
- 성능 50% 개선

### News Feed 광고
**Sponsored Stories**
- 자연스러운 피드 통합
- 모바일 최적화 포맷
- 타겟팅 정교화

**결과**:
- 2012 Q4: 모바일 광고 매출 23%
- 2013 Q4: 모바일 광고 매출 53%

## 📊 성과

### 재무 지표
| 연도 | 모바일 MAU | 모바일 매출 비중 | 주가 |
|------|------------|------------------|------|
| 2012 | 680M | 14% | $27 |
| 2013 | 945M | 45% | $55 |
| 2014 | 1.19B | 62% | $78 |
| 2015 | 1.44B | 77% | $105 |

### 제품 지표
- 모바일 체류 시간: 50분/일
- 모바일 광고 CTR: 데스크톱의 9.5배
- 앱 스토어 순위: 지속적 1위

## 🔑 성공 요인

### 1. CEO의 강력한 리더십
- 직접적인 위기 인정
- 명확한 방향 제시
- 빠른 실행

### 2. 과감한 인수
- 모바일 DNA 확보
- 경쟁 위협 제거
- 인재 영입

### 3. 개발자 우선 문화
- 해커톤 정례화
- 빠른 실험과 학습
- 데이터 기반 의사결정

## 💡 교훈

### PM을 위한 시사점
1. **플랫폼 전환 타이밍**: 사용자가 먼저 움직인다
2. **기술 부채 인정**: 잘못된 베팅 빠른 철회
3. **조직 정렬**: 전략과 구조 일치
4. **인수의 가치**: Build vs Buy 판단

### 장기적 영향
- 모바일 광고 시장 지배
- Instagram, WhatsApp 성장
- 메타버스 투자 기반
